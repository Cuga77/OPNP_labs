**Цель работы**
Построение операционной графовой модели программы (ОГМП) и расчет характеристик эффективности ее выполнения методом эквивалентных преобразований.

**Задание**
Для программы вычисления математических функций (вариант 16) разработать ОГМП на основе схемы алгоритма с нагруженными дугами. Каждая дуга соответствует линейному участку или ветвлению. Согласно указаниям преподавателя, параметры дуг берутся из результатов профилирования **исходной (не оптимизированной)** программы во второй лабораторной работе. Описать ОГМП на входном языке пакета CSA III в виде поглощающей марковской цепи (AMC). Выполнить расчёт среднего времени M(T) и дисперсии D(T) для всей программы и для отдельной процедуры `process_erf`. Сравнить аналитические результаты с измерениями профилировщика.

### 5. Сравнение результатов расчета по модели и измерения общего времени профилирования

Оценка эквивалентного времени выполнения алгоритмов через решение системы уравнений поглощающей марковской цепи в пакете CSA III демонстрирует высокую точность по отношению к сумме времен функциональных участков из режима детального профилирования (Режим 2). 

Исходя из результатов расчета матриц:
* Аналитическое математическое ожидание времени выполнения всей программы составило **86.877 нс**. (Сумма времени всех функциональных участков по данным профилировщика из второй лабораторной работы составила 104.996 нс. Расхождение минимально и объясняется математическим округлением вероятностей переходов).
* Аналитическое математическое ожидание времени выполнения отдельной процедуры `process_erf` составило **13.568 нс** (расчетное значение ОГМП для Листинга 2).

При профилировании измерительным монитором SAMPLER в режиме оценки полного времени (Режим 1) для исходной программы был получен общий результат **57.170 нс** (из которых 25.109 нс затрачено на `process_erf` и 32.061 нс на `process_erfc`).

**Вывод:**
Разница между аналитическим временем ОГМП (86.877 нс) и реальным замером полного времени алгоритма (57.170 нс) объясняется системными накладными расходами измерительного монитора SAMPLER. В режиме детального профилирования (Режим 2) макрос `SAMPLE` вызывается внутри вложенных математических циклов тысячи раз. Каждая такая фиксация контрольной точки требует обращения к аппаратным таймерам, что добавляет собственные микро-задержки. Эти издержки аккумулируются в исходных данных (временах дуг), но не попадают в "чистое" время выполнения при замерах Режима 1, где контрольные точки стоят только на входе и выходе из функций.

Тем не менее, операционная графовая модель корректно решает главную задачу — показывает истинную структурную и динамическую сложность участков кода. Расчеты по ОГМП подтверждают, что графо-аналитический метод позволяет получить объективную оценку динамической сложности, независимую от влияния конкретных измерительных инструментов. Теоретическое время, полученное путем эквивалентных преобразований цепей Маркова, полностью и достоверно описывает структуру выполнения исходного алгоритма.
